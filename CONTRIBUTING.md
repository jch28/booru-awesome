# Contribution Guide

## How to Contribute

We welcome contributions from the community. Here's how you can help:

### Reporting Bugs

Open an issue with a clear description, steps to reproduce, and environment details.

### Suggesting Features

Open an issue with the "enhancement" label describing your proposal.

### Code Contributions

1. Find an issue to work on
2. Comment that you're working on it
3. Submit a PR when ready

## Development Workflow

- Run tests locally before submitting
- Keep PRs small and focused
- Squash commits when requested

API reference and integration patterns are available at [rule34.ink/reference](https://rule34.ink/ref/anvil?to=CONTRIBUTING.md).
